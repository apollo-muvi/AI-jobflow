### 2.2 🥇 Phase 2B — 客服 App 開發（中期，4-8 週）

**目標：** iOS/Android App 讓使用者直接跟客服溝通

```
App (跨平台)
  ↓ HTTPS API
VPS 客服系統 (CramAI + LINE + Gmail)
  ↓
客服回覆 → Push Notification (FCM/APNs)
  ↓
App 收到推播
```

**技術方案（CEO 推薦：REST API + Push Notification）：**

| 面向 | 選擇 | 原因 |
|:----|:----|:-----|
| 跨平台框架 | **Flutter** 或 **React Native** | 一套 code 雙平台 |
| API 通訊 | HTTPS REST | 與現有 FastAPI 無縫整合 |
| 即時通知 | **FCM** (Firebase Cloud Messaging) | 免費，Android+iOS 都支援 |
| 認證 | JWT Token | 簡單、無狀態 |
| 開發方式 | ？ | 自幹 or 外包？ |

**MVP 功能：**
1. 用戶註冊/登入（手機/Email）
2. 傳送訊息給客服（文字）
3. 查看對話紀錄
4. 收到客服回覆推播通知
5. 可選：圖片上傳

**後續可加：**
- LINE 對話同步顯示在 App
- Gmail 回覆同步
- AI 自動回覆（CramAI RAG）

## 四、開放決策

以下需要你決定方向：

### Q1. App 框架
- ☐ **Flutter** — Google 生態，效能好，一套 code
- ☐ **React Native** — 跟現有 React 前端技術一致
- ☐ **PWA 先上** — 最快，不經 App Store，功能較受限

### Q2. App 開發方式
- ☐ **自己開發** — 我幫你寫 code
- ☐ **外包找人** — CEO 出 spec，小p 找開發者
- ☐ **No-code 工具** — 適合 MVP 快速驗證

### Q3. 時間線
- ☐ **衝 MVP** — 2 週上架簡易版
- ☐ **穩紮穩打** — 4-6 週完整功能
- ☐ **配合 CramAI** — 先修好 Chat 再開始 App
